## Summary

These architecture diagrams illustrate:

1. **System Architecture:** Multi-tier design with specialized indexes
2. **Indexing Pipeline:** Offline process to build rich indexes
3. **Query Processing:** Runtime flow from query to results
4. **Multi-Sheet:** Handling comparative queries across tabs
5. **Incremental Updates:** Real-time index updates on cell edits
6. **Scalability:** Production deployment with horizontal scaling

The hybrid approach leverages multiple specialized indexes (vector, keyword, formula, graph) orchestrated through a unified query processing pipeline, enabling accurate semantic search while maintaining explainability.
